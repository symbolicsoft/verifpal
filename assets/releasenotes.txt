Verifpal 0.51.0 is a correctness-focused release that tightens how the analysis engine reasons about passwords, checked assertions, and AEAD associated data — all on the back of excellent bug reports from the community:

- Password extraction has been rewritten from a static per-primitive "password_hashing" table into a dynamic verifiable-guess check: a password is obtainable only if no ancestor primitive protects its position and the attacker knows every sibling argument needed to verify a guess. This fixes surprising leaks inside CONCAT, and produces accurate results across ENC, MAC, AEAD_ENC, SIGN, BLIND, and SHAMIR_SPLIT.
- Checked ASSERT(...)? now genuinely halts the principal: every `leaks` declaration is recorded as a LeakEvent with its principal and declared_at, and any leak that sits after a failed check in the same execution branch is suppressed. This eliminates a class of false-positive active attacks.
- AEAD_ENC no longer passively reveals its associated data on the wire. Associated data is an input to AEAD, not part of the ciphertext, and leaking it by default was blocking legitimate queries (e.g. passwords used as AD).
- The TUI deduction ticker now correctly labels multi-output primitives (HKDF, SPLIT, SHAMIR_SPLIT) that previously rendered as "First" or "Second".

None of the changes affect the Verifpal modeling language — every existing model still parses and runs, producing results that are at worst the same and in several cases strictly more accurate. 152 tests pass (up from 147).

Thanks to Cédric Picard (@cym13) for issues #11, #12, and #13, and to @EFCCWEB3 for issue #10.
